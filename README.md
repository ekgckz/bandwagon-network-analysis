## 一、先把搬瓦工的"线路家族"理清楚

搬瓦工官网对线路的描述其实写得挺直白——中国方向有三大运营商（电信、联通、移动），每家又分好几个等级，从便宜到贵再到很贵。搬瓦工能卖这么多年的核心原因之一，就是它在中国方向上做了相当细致的线路分层。我们把这几条线按"等级从低到高"捋一遍。

### AS4134 ChinaNet / 163：最便宜也最堵的"国民线路"

这是国内方向最基础的国际出口，几乎所有云厂商都在用。优点是便宜、容量大、抗 DDoS 能力还行；缺点就是晚高峰会堵，丢包率能飙到 30% 以上。搬瓦工的 KVM 常规套餐（弗里蒙特 FMT、新泽西 USNJ、纽约 USNY_2、荷兰 EUNL_3、加拿大 CABC_1 等）走的基本就是这一类普通国际线路。预算党拿来学习 Linux、跑脚本、做备用节点是够用的，但如果你是面向国内访客建站，晚高峰的体验会劝退你。

### CN2 GT（AS4809 Global Transit）：曾经的"中端优化"，如今更像是历史名词

CN2 GT 当年是用来缓解 163 拥堵的，价格比 163 贵一截，定位中端。但从 2019 年开始，CN2 GT 本身也变得跟 163 差不多堵了。搬瓦工早期的 DC2、DC3、DC8 机房让很多老用户接触过这条线，但今天再盯着"CN2 GT"去买已经意义不大——搬瓦工当前产品线里，KVM 常规套餐就不应该再按 CN2 GT 来理解。新用户没必要再纠结这个标签，把它当历史了解一下就行。

### CN2 GIA（AS4809 Global Internet Access）/ CTGNet（AS23764）：电信方向的"天花板"

这是中国电信方向最高等级的优化线路。搬瓦工官方在洛杉矶运营着 8 条 10Gbps 的 CN2 GIA/CTGNet 链路，分布在两个机房。它的稳定性是真的好，丢包低、晚高峰扛得住，适合做 Web 会议、VoIP、面向国内的内容服务、在线游戏这种对稳定敏感的场景。缺点也明确：贵（CN2 GIA 单兆 IP transit 价格能到 $120/Mbps），容量有限，对 DDoS 不友好。CTGNet 是电信后来加的新选项，搬瓦工官方说它在实际表现上和 CN2 GIA 基本等效。

### CMIN2（AS58807）与 China Unicom Premium（AS10099）：移动和联通方向的优化

CN2 GIA 是电信方向的事，那移动和联通用户呢？搬瓦工在 USCA_9（洛杉矶 DC9）机房把去往中国的流量分发给三家运营商：电信走 CN2 GIA、移动走 CMIN2、联通走 China Unicorn Premium（俗称 9929）。这就是为什么 DC9 被很多人称为"三网优化"机房——不管你用户用哪家宽带，都能踩到对应的优化线路。

> 一句话总结等级：163 < CN2 GT < CN2 GIA ≈ CTGNet，而搬瓦工的 CN2 GIA-E 套餐（DC6/DC9）就是把 CN2 GIA + CMIN2 + CUP 这三条优化线路打包在一起的高端方案。
